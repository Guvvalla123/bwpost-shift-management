====================================================
  BWPOST Shift Management — JSX Files Overview
====================================================
Date: 04 March 2026

----------------------------------------------------
AUTH
----------------------------------------------------

Login.jsx
  - Login form with email/password validation
  - Calls POST /api/users/login
  - On success: fetches /api/users/me, updates AuthContext (name, email, role, photo)
  - Redirects managers → /manager/dashboard, employees → /employee

Register.jsx
  - Registration form for new users
  - Calls POST /api/users/register

----------------------------------------------------
CORE LOGIC
----------------------------------------------------

main.jsx
  - Entry point of the entire app
  - Wraps the app inside AuthProvider and BrowserRouter
  - Without this, no route or auth context would work

App.jsx
  - Defines ALL routes of the application
  - /login, /register → public pages
  - /manager/* → protected (only managers can access)
  - /employee/* → protected (only employees can access)
  - Uses ProtectedRoute to enforce this

context/AuthContext.jsx
  - GLOBAL auth state: stores { id, role, username, email, profileImage }
  - On app load: checks session via /api/users/me; if expired, tries refresh token
  - Provides: user, loading, login(), logout(), updateUser()
  - updateUser() lets any component silently merge updates (e.g. new photo)
    without forcing a full page reload

components/ProtectedRoute.jsx
  - Wraps any route that needs authentication
  - Shows a spinner while auth is loading
  - If not logged in → redirects to /login
  - If wrong role → redirects away
  - Protects all manager and employee routes

----------------------------------------------------
KEY CONCEPT: AuthContext is the heart of the app
----------------------------------------------------

Every component reads from AuthContext:
  - user.username  → shown in greetings, sidebar, navbar
  - user.email     → shown in profile dropdown, settings
  - user.role      → used by ProtectedRoute to allow/deny access
  - user.profileImage → shown as avatar everywhere

updateUser({ profileImage: newUrl }) called in Settings:
  → Sidebar avatar updates instantly
  → Navbar avatar updates instantly
  → No page refresh needed
